# Скрипт заполнения базы данных

Этот скрипт заполняет базу данных тестовыми данными для разработки и тестирования.

## Что создается:

1. **1 администратор**
   - Логин: `admin`
   - Пароль: `admin123`

2. **30 номеров** (5 этажей × 6 номеров на этаже)
   - Номера с 101 по 130
   - Разные типы: SINGLE, DOUBLE, TRIPLE
   - Цены: 2500₽, 3500₽, 4500₽ соответственно

3. **10 работников** (по 2 на каждый этаж)
   - Логины: `worker1`, `worker2`, ...
   - Пароль: `password123`

4. **Расписание уборки**
   - Каждый работник убирает свой этаж в определенные дни недели

5. **50 клиентов**
   - Логины: `client1`, `client2`, ...
   - Пароль: `password123`
   - Случайные даты заезда (в прошлом и будущем)
   - Случайные города из списка российских городов

6. **Счета**
   - Создаются для 30% клиентов
   - Случайные даты выдачи

## Запуск скрипта

### Способ 1: Через Gradle

```bash
cd backend
./gradlew run --args="seed"
```

Или если у вас Windows:
```bash
cd backend
gradlew.bat run --args="seed"
```

### Способ 2: Напрямую через main функцию

1. Убедитесь, что база данных запущена и доступна
2. Проверьте параметры подключения в файле `SeedDatabase.kt`:
   ```kotlin
   Database.connect(
       url = "jdbc:postgresql://localhost:5432/hotel_db",
       driver = "org.postgresql.Driver",
       user = "hotel_admin",
       password = "orf.12014319N"
   )
   ```

3. Запустите main функцию в `SeedDatabase.kt`

### Способ 3: Через IDE

1. Откройте файл `backend/src/main/kotlin/scripts/SeedDatabase.kt`
2. Запустите функцию `main()`

## Важно

- Скрипт проверяет существование администратора и не создает дубликаты
- При повторном запуске будут созданы новые клиенты, работники и номера
- Убедитесь, что база данных инициализирована (таблицы созданы)

## Данные для входа

После запуска скрипта вы можете войти в систему:

- **Администратор**: `admin` / `admin123`
- **Клиенты**: `client1`, `client2`, ... / `password123`
- **Работники**: `worker1`, `worker2`, ... / `password123`
